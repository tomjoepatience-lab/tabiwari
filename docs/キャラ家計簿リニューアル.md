# リニューアル構想「マネコ家計簿」— 3Dキャラクターと暮らす家計簿

作成: 2026-07-07

## コンセプト

家計簿の主役を「表」から「キャラクター」に変える。
ホーム画面には 3D キャラクター **マネコ**（マネー＋ネコの妖精）が住んでいて、
買ったものを食べたり・持ったり・身につけたりして反応する。
使いすぎのときは**叱らず、あくまでアドバイス**として声をかけてくる。

> 例:
> - ラーメンが月に何度も続く →「今月ラーメン5回目だにゃ…最近ちょっと丸くなってきた気がする。来週は2回までにしない?」
> - 飲み会が続く →「うぅ、肝臓がきついにゃ〜。次の週末はおうちでゆっくりしよ?」
> - 日用品・家賃・交通などの消耗品/固定費 → 反応はするが警告はしない。

既存アプリ（TabiWari）の良いところは残す:
- レシート取り込み（Claude vision OCR で明細下書き）
- 月間サマリー（カレンダー・レポート）
- 地図（買い物・思い出のピン）と思い出写真・アルバム
- 割り勘・グループの仕組みは**バックエンドに温存**（UIは個人利用を最優先に再構成）

## キャラクター「マネコ」

- 全身のある 3D ちびキャラ。トゥーンシェーディング＋輪郭線でアニメ調に。
- 見た目最優先: 大きな頭・大きな瞳・ω口・ほっぺ・しっぽ。子供も大人も好きになれる路線。
- **体型が変わる**: 食べすぎ月はちょっと丸くなる（chubby係数）。節度ある月はスリムに戻る。
- **衣装**: 帽子・マフラーなどの着せ替え。節約達成などで増やせる余地（ごほうび設計）。
- **反応**: 支出のカテゴリ/明細キーワードで、食べる・飲む・掲げる・着る等のアニメーション＋ひとこと。
- 放置すると眠る、タップするとよろこぶ、時間帯であいさつが変わる。
- 実装: three.js をプロシージャル生成（外部モデル不使用・全部コードで造形）。`src/client/character.ts`

## おとなモード本格機能化（2026-07-07 第3弾）

1アカウント・1デバイス前提に簡素化し、記録〜分析を大人向けに詰めた:
- **プロジェクト/グループUIを廃止**（バックエンドは温存・自動プロビジョニングのみ）。せってい＝モード切替/予算収入/ログアウトだけ
- **詳細きろくがデフォルト**: お店・日付・場所（地図ピッカー）・思い出写真（最大3枚・縮小してBYTEA保存）・複数明細
- **自動ジャンル分け**: `src/shared/genre.ts`（品名キーワード優先→店名フォールバック、11ジャンル）。例: スーパーでも「アイス/酒」は嗜好品。サーバー保存時に確定、明細タップ→ジャンルシートでいつでも手直し（PUT /api/items/:id/genre）。レポート/ホームのつかいみちは items.genre 集計
- **見やすい明細**: レシートは羅列ではなくジャンル別グループ＋小計＋合計＋写真の「明細カード」（records.ts receiptCard）
- **地図**: レポートに買い物マップ。場所つき記録にピン、タップで明細カード＋写真
- **カレンダー**: 日付タップでカードがポップして明細・写真（下に出さない）
- **マネコ分析係**: insights.ts が「放置しがちだが大切」な情報を生成（予算ペース/前月同時期比/トップジャンル/嗜好品比率/通いすぎの店/最大の買い物/記録の空白）→吹き出しでローテーション
- **月初サマリー**: その月はじめて開いたとき先月のまとめ（合計/前月差/トップ3ジャンル/最大の買い物/記録日数/ひとこと）をモーダル表示（user_settings.last_summary_shown で一度きり）
- 学び: `Number(null)===0` で場所なし記録に lat=0,lng=0 が入る「幽霊ピン」→ null 先判定で修正。モーダルは Leaflet(z〜1000) より上の z-index 3000

## 忠実移植方針（2026-07-07 改訂）

ユーザーFB「デザインを忠実に再現して」を受け、**モック（.dc.html）のHTML/CSSをそのまま移植**する方式に全面変更:
- 画面は 402×840 の絶対配置キャンバス（`src/client/phone.ts`、狭い画面は等倍縮小）
- マネコは3Dをやめ、モックの **CSS描画のオレンジしま猫**（琥珀の縦長瞳・鈴つき赤首輪・おなかの¥コイン・招き手 mbeckon・しっぽ mwag・回転コイン mspin）を verbatim 移植
- 2a: 空グラデ・光の帯・雲・石畳＋金の道(clip-path)・パンや/ゲーム/おかしや・目標ショーケース（ゲーム機＋台座）・買ったもの小物（ロリポップ/ゲームソフト）・金縁HUD・Lvリング(conic)・チャレンジカード・エッジボタン・ゲームふう下部ナビ
- 3a: 墨色ヒーローカード・覗きミニマネコ＋吹き出し・つかいみち(grid 76px/1fr/72px)・最近の記録・白タブバー(中央56px墨色＋金の＋)
- アニメは全て CSS keyframes（mfloat/mspin/mwag/mbeckon/msparkle/mfall/mcloud/mjump…style.css へ移植）。タップでマネコが mjump
- 動的注入は数値・文言のみ（残高/ハート/Lv%/チャレンジ状態/目標名と進捗/買ったものラベル/カテゴリ行/最近の記録）。ユーザー文字列は esc() でエスケープ

## 2アプリ構成（2026-07-07〜・デザインハンドオフ 2a/3a 準拠）

1つのアカウントで**モード切替**できる2つのアプリとして実装
（デザイン: `claude/マネコのキャラクター・ホーム画面デザイン/design_handoff_maneko_home/`）。

### 🏘️ こどもモード「マネコタウン」（2a）
- 3Dの街（金の道・パンや・ゲームショップ）にマネコが立ち、買ったものが道ぞいの看板になって浮かぶ
- **おさいふ**: スタート額＋おこづかい（incomes）− 支出 − ちょきん で自動計算
- **コイン**: きょうのチャレンジ（その日はじめての記録で+5）・レベルアップ+10・目標達成+50。🎁プレゼント（30コイン）で衣装ガチャ
- **XP/Lv**: 記録+10 / チャレンジ+5 / ちょきん+20 / 目標達成+100。Lv = 1+floor(√XP/5)。LvリングをHUDに表示
- **ごきげん（♥×4）**: 3が基準。連続記録3日+1、使いすぎ警告−1、おさいふマイナス−1（最低1・叱らない）
- **もくひょう/ちょきん箱**: ほしいもの（絵文字＋目標額）に入金 → 進捗バー。達成でボーナス＋お祝い
- タブ: ホーム / レポート / きろく（中央の金コイン）/ ちょきん / メニュー

### 💼 おとなモード「マネコ家計簿」（3a）
- 紙色 #F7F4EE ×墨色カード。マネコは残高カードから頭だけ覗く応援キャラ（peek バリアント）
- **今月あと使える** = 予算 − 支出、使用率バー、収入/支出。予算ペースで応援チップの文言が変わる
- **つかいみち**: 今月のカテゴリ別バー（金/緑/茶/青のトークン色）
- **最近の記録**: カテゴリ頭文字アイコン＋店名＋きょう/きのう表示。タップで覗きマネコが反応
- タブ: ホーム / レポート / きろく（墨色の＋）/ ちょきん / せってい（予算・収入・プロジェクト・グループ）
- 取捨選択: 旅行割り勘・OCR・アルバム・グループは「せってい → プロジェクト」に温存（こどもモードでは非表示）

### 共通基盤
- `user_settings`（mode/予算/収入/おこづかい/コイン/XP/衣装/チャレンジ日）・`savings_goals`・`incomes` テーブル
- `GET /api/overview`（残高・今月収支・目標・チャレンジをまとめて返す）・`PUT /api/settings`・
  `POST /api/goals` `/goals/:id/deposit` `/incomes` `/present`
- 記録（`POST /api/expenses/quick`）がごほうび（XP/コイン/チャレンジ/レベルアップ）を返し、ホームでマネコがお祝い
- 初回ログインはモード選択（こども/おとな）。フォントは M PLUS Rounded 1c
- renderHome は世代番号（epoch）ガードで古い非同期描画を破棄（保存→即タブ移動のレース対策）

## 画面構成（ホームをタブ化・シンプルさ最優先）

| タブ | 内容 |
|---|---|
| 🐱 ホーム | マネコのステージ（3D）＋吹き出し＋今月の支出＋最近の買い物（タップでマネコが反応）。背景に最近買ったものが浮かび、金貨が回る |
| ✏️ きろく | **ワンタップ記録**。品名＋金額（＋カテゴリチップ）だけで保存 → ホームに戻ってマネコがお祝い反応。グループ・プロジェクト・メンバーはサーバーが自動作成（`POST /api/expenses/quick`） |
| 📅 カレンダー | 月間カレンダー。日別の支出合計、タップで明細 |
| 📊 レポート | 月次推移（6ヶ月）・カテゴリ別・プロジェクト別 |
| ⚙️ その他 | プロジェクト一覧/作成・グループ管理・ログアウト（詳細機能はここに集約） |

詳細な記録（レシートOCR・割り勘・地図ピッカー）は従来どおりプロジェクト詳細で行う（きろくタブから誘導）。
プロジェクト詳細の下部タブ先頭に「🐱 ホーム」を常設し、トップへ戻ると必ずマネコのホームを表示する（迷子防止）。

## 信頼性（2026-07-07 の学び）
- Neon の一時的な接続断で attachUser のセッション照会が失敗すると 401 → 画面全体が死んでいた。
  対策: attachUser はリトライ、直近支出の取得失敗でもホームは表示、route エラー時は「再読み込み/ホームへ」ボタンを出す。
- WebGL が使えない環境では 3D の代わりに絵文字マネコのフォールバックを表示（ホームは壊さない）。

## アドバイスエンジン（`src/client/advice.ts`）

- 直近30日の明細を解析（`GET /api/expenses/recent`）。
- キーワード×カテゴリで検出: ラーメン/外食、酒・居酒屋・飲み会、カフェ、スイーツ、コンビニ。
- 日用品（洗剤・トイレットペーパー等）・固定費（家賃・光熱・交通）は**警告対象外**。
- 出力: 気分(mood)・体型(chubby)・吹き出しメッセージ群・ほめ言葉（節度ある月/記録の継続=ストリーク）。
- ルールベースで開始。将来は AI 相談（Claude API）に拡張可能な口を残す。

## 決済連携の方針（フェーズ2）

PayPay・カード会社に個人向け公開 API は無いため、現実解は:
1. **CSV インポート**（カード明細・PayPay の取引履歴CSV）→ 明細としてまとめて取り込み、マネコが反応
2. メール転送パース（利用速報メールを転送→サーバで解析）— 将来枠
3. 取り込み時に「新しい決済があったにゃ！」と通知風の演出

## 地図ルート再生（フェーズ2）

既存の地図ピンを時系列に接続し「ルート」として再生:
- ルートを選ぶと、マーカー（マネコアイコン）がピンを順にたどる
- 各ピンで立ち止まり、そのときの店名・金額・思い出写真・ひとことを吹き出しで表示

## 技術メモ

- three.js は npm 依存で esbuild にバンドル（CDN 不使用）。
- キャラはプリミティブ合成＋MeshToonMaterial＋反転ハル輪郭線。表情は目メッシュ＋口のCanvasテクスチャ。
- 吹き出しは HTML オーバーレイ（日本語が滲まない）。
- 既存の認証・グループ・割り勘・OCR・アルバムは変更しない（非破壊）。
