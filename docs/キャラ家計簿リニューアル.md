# リニューアル構想「マネコ家計簿」— 3Dキャラクターと暮らす家計簿

作成: 2026-07-07

## コンセプト

家計簿の主役を「表」から「キャラクター」に変える。
ホーム画面には 3D キャラクター **マネコ**（マネー＋ネコの妖精）が住んでいて、
買ったものを食べたり・持ったり・身につけたりして反応する。
使いすぎのときは**叱らず、あくまでアドバイス**として声をかけてくる。

> 例:
> - ラーメンが月に何度も続く →「今月ラーメン5回目だにゃ…最近ちょっと丸くなってきた気がする。来週は2回までにしない?」
> - 飲み会が続く →「うぅ、肝臓がきついにゃ〜。次の週末はおうちでゆっくりしよ?」
> - 日用品・家賃・交通などの消耗品/固定費 → 反応はするが警告はしない。

既存アプリ（TabiWari）の良いところは残す:
- レシート取り込み（Claude vision OCR で明細下書き）
- 月間サマリー（カレンダー・レポート）
- 地図（買い物・思い出のピン）と思い出写真・アルバム
- 割り勘・グループの仕組みは**バックエンドに温存**（UIは個人利用を最優先に再構成）

## キャラクター「マネコ」

- 全身のある 3D ちびキャラ。トゥーンシェーディング＋輪郭線でアニメ調に。
- 見た目最優先: 大きな頭・大きな瞳・ω口・ほっぺ・しっぽ。子供も大人も好きになれる路線。
- **体型が変わる**: 食べすぎ月はちょっと丸くなる（chubby係数）。節度ある月はスリムに戻る。
- **衣装**: 帽子・マフラーなどの着せ替え。節約達成などで増やせる余地（ごほうび設計）。
- **反応**: 支出のカテゴリ/明細キーワードで、食べる・飲む・掲げる・着る等のアニメーション＋ひとこと。
- 放置すると眠る、タップするとよろこぶ、時間帯であいさつが変わる。
- 実装: three.js をプロシージャル生成（外部モデル不使用・全部コードで造形）。`src/client/character.ts`

## 親子アカウント連携（将来構想・設計のみ）

**目的**: 親（おとなモード）が子（こどもモード）を見守り、おこづかい・お手伝いポイントを送れるようにする。

### 連携モデル
- `account_links (id, parent_user_id, child_user_id, status: pending/active, created_at)` を追加（1親:N子。子側は親1人まで）
- 連携手順: 親が「連携コード」（8桁・randomBytes・10分有効）を発行 → 子の端末のメニューで入力 → 双方に確認表示 → active
- 解除: どちらからでも可能（データは残る。閲覧・送金だけ止まる）

### 連携中にできること
- **親→子の閲覧**: 子の おさいふ残高・今月の記録・目標進捗を親のメニュー「こどもの様子」で読み取り専用表示（GET /api/children/:id/overview。親権限チェック必須）
- **おこづかい送金**: 親が金額を指定 → 子の incomes に「おこづかい（パパ/ママから）」を作成。子のホームでお祝い演出
- **お手伝いポイント**: 親がお手伝いメニュー（例: おふろそうじ=50pt）を登録 → 子が「やった！」→ 親が承認 → ポイント付与。ポイントは `chore_points` に貯まり「¥換算しておさいふへ」or「コインに交換」
- **モード制限**: 連携中の子アカウントは おとなモードへの切替ボタンを非表示（サーバー側でも PUT /api/settings mode=adult を 403）。未連携なら従来どおり自由に切替可

### 実装順（着手時）
1. account_links テーブル＋連携コードAPI＋メニューUI（発行/入力/解除）
2. 親の「こどもの様子」読み取り画面
3. おこづかい送金（incomes 連携＋演出）
4. お手伝いポイント（登録/申請/承認/交換）

## PWA化＋スクロール修正（2026-07-08）

### PWA（ホーム画面に追加できる）
- `public/manifest.webmanifest`（name=マネコ家計簿, short_name=マネコ, display=standalone, portrait, theme_color=#E8B62B, background_color=#F6EFE2, icons 192/512 "any maskable"）
- `public/sw.js`（Service Worker）: app shell を precache してオフライン起動可。`/api/` はキャッシュしない（常に最新）。ページ遷移=network-first→失敗時cache、静的=stale-while-revalidate、フォントのみ別オリジンSWR、地図タイルは素通し。`activate` で旧キャッシュ削除。
- `public/icons/*.png`（マネコのオレンジしま猫＋¥コインを SVG から生成）。apple-touch-icon 180。
- `index.html`: manifest/theme-color/apple-touch-icon/apple-mobile-web-app-*/viewport-fit=cover ＋ SW 登録 inline script。`?v=6` に更新（sw.js の CORE と一致）。
- 検証: manifest 200(application/manifest+json)、SW active、オフライン再読み込みで app shell 復帰、installability 要件充足を headless で実証。

### スクロール不具合の修正（ちょきん/きろく等が下までスクロールできない）
- 真因: wrapPhone のHTMLに **402×840固定の内側div** があり、それが `.pc-scroll(inset:0)` の包含ブロックになって高さが840に固定。キャンバスを縮めても内容がその高さのままで、下端（貯金目標の作成フォーム・記録ボタン）がタブバーの裏に隠れ、さらに body との二重スクロールになっていた。
- 修正: ①`phone.ts` に fillHeight オプション。きろく/レポート/ちょきん/せっていは transform:scale をやめ、キャンバス高さを visualViewport にフィット（横は幅そのまま＝文字は縮まない）。②wrapPhone の内側divを撤去し `.pc-scroll` とナビを `.pc-canvas` 直下に。これで `.pc-scroll` が canvas に追従し内側スクロール1本になる。ホーム（kids/adult）は絶対配置(402×840)なので fillHeight=false のまま。
- 学び: transform を持つ要素は position:absolute の子の包含ブロックにならない。二重div＋固定高さ＋transform が絡むと `inset:0` が意図の親を取れない（offsetParent で発覚）。

## おとなモード本格機能化（2026-07-07 第3弾）

1アカウント・1デバイス前提に簡素化し、記録〜分析を大人向けに詰めた:

## 決済連携の方針（フェーズ2）

## 技術メモ

- three.js は npm 依存で esbuild にバンドル（CDN 不使用）。
- キャラはプリミティブ合成＋MeshToonMaterial＋反転ハル輪郭線。表情は目メッシュ＋口のCanvasテクスチャ。
- 吹き出しは HTML オーバーレイ（日本語が滲まない）。
- 既存の認証・グループ・割り勘・OCR・アルバムは変更しない（非破壊）。
