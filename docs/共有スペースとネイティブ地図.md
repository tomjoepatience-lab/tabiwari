# 共有スペース・Expoネイティブ地図

## リリース構成

- iOSの正式クライアントは `expo/`
- 家計簿本体・認証・APIは従来どおりWebアプリ
- 地図の表示と場所選択はWebViewからExpoへメッセージを送り、`react-native-maps`で表示
- iOSでは `PROVIDER_GOOGLE` を指定し、Google Maps SDK for iOSを使用

## リリース前の環境設定

サーバー:

```text
PUBLIC_APP_URL=https://tabiwari-mu.vercel.app
APPLE_TEAM_ID=<Apple DeveloperのTeam ID>
```

Expo / EAS:

```text
GOOGLE_MAPS_IOS_API_KEY=<Google Maps SDK for iOSのAPIキー>
```

Google Cloud側ではキーを次の条件に制限する。

- Application restriction: iOS apps
- Bundle identifier: `com.tomjo.maneko`
- API restriction: Maps SDK for iOS

Web用の `GOOGLE_MAPS_API_KEY` とは分けることを推奨する。

## DB反映

デプロイ前に最新の `src/db/schema.sql` を既存DBへ適用する。すべて
`IF NOT EXISTS` / `ADD COLUMN IF NOT EXISTS` を使っており、既存ユーザーを残して移行できる。

主な追加:

- `users.email`
- `users.display_name`
- `group_invites`
- `user_settings.active_group_id`

## 招待URL

形式:

```text
https://tabiwari-mu.vercel.app/invite/<token>
```

- 有効期限7日
- 最大10回参加
- 未ログイン時はログイン／登録後に自動参加
- インストール済みiOSではUniversal LinksでExpo版を開く
- 未インストール時はWebの参加画面を開く

Universal Linksには、サーバーの `APPLE_TEAM_ID` とExpoの
`associatedDomains` の両方が必要。
